Partir d'un T1, sans DICOM

Vous avez déjà un T1 en NIfTI (dé-identifié, ou issu d'un autre logiciel) et pas les DICOM d'origine. Le pipeline part directement de cette image.

Un seul argument change

On donne t1_path au lieu de dicom_dir. Exactement un des deux est requis — fournir les deux (ou aucun) lève une erreur immédiate.

result = reconstruct_sources(
    subject="sample", output_dir="D:/derivatives",
    t1_path="D:/anat/sample_T1w.nii.gz",  # <- au lieu de dicom_dir
    eeg_file="D:/eeg/sample.edf",
    digitization="D:/dig/sample.elc",
    simnibs_bin_dir="C:/Users/me/SimNIBS-4.5/bin",
    events="find", event_id={"aud_l": 1},
)

La conversion DICOM→NIfTI est simplement sautée ; le T1 est copié tel quel et alimente charm. Toutes les étapes suivantes sont identiques.

Un T2 (t2_path) améliore nettement la segmentation du crâne si votre protocole en comporte un. L'anonymisation DICOM n'a plus lieu d'être ici — assurez-vous que le T1 est déjà dé-identifié avant de le partager.
